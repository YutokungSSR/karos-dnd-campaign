# กฎการพัฒนาโปรเจกต์ Karos D&D Campaign

ไฟล์นี้มีผลกับทุกไฟล์และทุกโฟลเดอร์ใน Repository นี้ ผู้ช่วยพัฒนาทุกคนต้องปฏิบัติตามกฎต่อไปนี้ก่อนเริ่มงาน ระหว่างพัฒนา และก่อนส่งงาน

## การใช้ Git และ Branch

- ห้ามแก้ไข Commit หรือ Push เข้า `main` โดยตรง
- ทุกงานต้องเริ่มจาก `origin/main` ล่าสุด แล้วสร้าง branch งานใหม่ก่อนแก้ไขไฟล์
- ตั้งชื่อ branch ตามประเภทงาน:
  - `feature/ชื่อฟังก์ชัน` สำหรับฟังก์ชันใหม่
  - `fix/ชื่อปัญหา` สำหรับการแก้บั๊ก
  - `chore/ชื่องาน` สำหรับงานดูแลโปรเจกต์หรือการตั้งค่า
- ก่อนแก้ไขต้องตรวจ `git status` และอ่านโครงสร้างโปรเจกต์ที่เกี่ยวข้อง
- Push ได้เฉพาะ branch งาน แล้วสร้าง Pull Request ไปยัง `main`
- ห้าม Merge Pull Request จนกว่าผู้ใช้จะสั่งอย่างชัดเจน
- ห้าม `force push`, `reset --hard`, ลบข้อมูล หรือทำการเปลี่ยนแปลงที่กู้คืนได้ยากโดยไม่ได้รับอนุญาต

## ความปลอดภัยของข้อมูล

- ห้ามเปิดเผยหรือ Commit API key, password, token, ไฟล์ `.env`, ค่า Environment Variable หรือข้อมูลลับทุกชนิด
- ห้ามแก้ Environment Variables ของ Production โดยไม่ได้รับอนุญาตจากผู้ใช้
- ก่อน Commit ต้องตรวจ diff และยืนยันว่าไม่มีข้อมูลลับหรือไฟล์ที่ไม่เกี่ยวข้อง

## การติดตั้งและทดสอบ

- รัน `npm install` เมื่อจำเป็น เช่น ยังไม่มี dependencies หรือมีการเปลี่ยนแปลง dependencies
- หลังแก้ไขต้องรัน `npm run lint` และ `npm run build` ก่อน Push
- หากการทดสอบไม่ผ่าน ห้ามอ้างว่างานเสร็จ และห้าม Merge
- หากไม่สามารถรันทดสอบได้ ต้องแจ้งเหตุผลและผลกระทบตามจริง

## การส่งมอบงาน

- ต้องสรุปไฟล์ที่แก้ เหตุผลของการแก้ ผลการทดสอบ และผลกระทบต่อระบบ
- อธิบายผลลัพธ์ให้ผู้ใช้เป็นภาษาไทยแบบเข้าใจง่าย
- รักษาดีไซน์แฟนตาซีดำ–ทองและพฤติกรรมเดิมของระบบ เว้นแต่ผู้ใช้สั่งให้เปลี่ยน
